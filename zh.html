集感知、实时智能、可视化与人机智能交互于一体的综合平台

CPS Smart Home V2 是一个完整的网络物理系统，将分布式传感器、实时数据服务、交互式仪表板、安全用户访问和本地运行的 AI 助手连接在一起。系统采集多源 JSON 数据，通过中央 API 处理并实时推送，在前端展示系统状态与分析结果，同时支持智能对话，且不会把本地 AI 服务直接暴露给浏览器。本指南涵盖整个平台的架构、安装、配置、运行、测试与部署。

前端介绍

前端将多个独立的智能家居感知数据源集中显示，主要用户包括学生、教师和实验室演示人员。范围包括中英文界面、学科提升页面、身份验证、传感器图表、设备在线/离线状态、事件面板、视频占位界面和 CPS 对话。

前置要求

Node.js建议 22.x支持 ≥22 且 <25
Yarn安装 Node.js 后安装使用 yarn.lock
Git最新稳定版本用于获取源码
浏览器Chrome、Edge 或 Firefox最新稳定版本
Supabase项目 URL 和发布密钥用于托管身份验证
后端服务本地或已部署地址用于实时数据和对话

安装 Node.js、Git 和 Yarn

  1. 安装 Node.js 22 LTS

    从 Node.js 官方网站下载适用于当前操作系统的安装程序。使用默认安装选项；npm 会随 Node.js 一同安装。安装完成后关闭并重新打开终端。

  2. 安装 Git

    从 Git 官方网站下载安装程序并使用默认选项。Windows 用户可以使用 PowerShell 或 Git Bash。

  3. 检查 Node.js、npm 和 Git
    node --version
npm --version
git --version

    Node.js 应显示 22.x 版本。

  4. 安装 Yarn
    npm install --global yarn
yarn --version

    如果不能立即识别 yarn 命令，请重新打开终端。

安装

  1. 下载前端项目

    使用 Git 克隆仓库：

    git clone https://github.com/Zamidmustefa/cps_smart_home_frontend_v2.git
cd cps_smart_home_frontend_v2

    如果没有使用 Git，请打开前端仓库，选择 Code → Download ZIP，解压后在该文件夹中打开终端。

  2. 安装全部前端依赖

    使用锁文件安装经过测试的依赖版本。

    yarn install --frozen-lockfile
  3. 创建前端环境文件
    # Windows PowerShell
Copy-Item .env.example .env

# macOS 或 Linux
cp .env.example .env

    使用文本编辑器打开 .env，替换 Supabase 占位值。如果前后端在同一台电脑运行，请使用下方列出的本地后端地址。

配置与设置

变量是否需要说明
PORT是HTTP 与 WebSocket 端口，默认 7020。
AGENT_TOKEN对话需要仅本地 Ollama 智能体使用的私密令牌。
FRONTEND_URLS是CORS 允许的前端来源，以逗号分隔。
DATA_CHANNEL_API_KEY生产环境保护传感器 POST；为空时允许本地匿名写入。
DATA_CHANNEL_HISTORY_LIMIT否每通道最多记录数，默认 25。
DATA_CHANNEL_MAX_HISTORY_BYTES否每通道内存上限，默认 5 MB。
CLIENT_TIMEOUT_MS否对话超时，默认 120000 毫秒。
KOKORO_ENABLED否512 MB 云实例应保持 false。

最小本地 .env 配置

NODE_ENV=development
PORT=7020
AGENT_TOKEN=replace-with-a-long-random-token
CLIENT_TIMEOUT_MS=120000
FRONTEND_URL=http://localhost:3000
FRONTEND_URLS=http://localhost:3000,http://localhost:5173
PUBLIC_BACKEND_URL=http://localhost:7020
DATA_CHANNEL_API_KEY=
KOKORO_ENABLED=false
KEEP_ALIVE_ENABLED=false

仅在本地开发时可以将 DATA_CHANNEL_API_KEY 留空；生产环境必须设置强密钥。完整的 .env.example 还包含可选的 Ollama 智能体、搜索、语音服务、历史记录和 Kokoro 设置。

不要提交密钥：真实 .env、数据通道 API 密钥、Supabase 私密凭据和智能体令牌均不得写入文档或提交到源码仓库。

智能家居数据通道

每个通道互相独立。POST 接受 JSON 对象或数组并返回 202 Accepted。数据保存在内存中，后端重启后会清空。

CSI – WiFicsi-wifi
身体体征body-vitals
照明与声学lighting-acoustic
人机交互hic
视线与视觉gaze-vision
操作路径用途
POST/api/data-channels/:channel接收一条新 JSON 数据。
GET/api/data-channels/:channel/latest返回最新数据。
GET/api/data-channels/:channel/history?limit=20按从旧到新的顺序返回历史。
SSE/api/data-channels/:channel/stream成功写入后立即推送。

请求示例

curl -X POST http://localhost:7020/api/data-channels/body-vitals \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_INGESTION_KEY" \
  -H "X-Source-Id: student-laptop-01" \
  -d '{"timestamp":"2026-07-19T12:00:00Z","heart_rate_bpm":78}'

学生文件发送程序

# Node.js 20+
node examples/send-json-file.js data.json

# Python 3，不需要第三方包
python examples/send_json_file.py data.json

X-Source-Id 不是密码。它是显示发送设备的可读名称，例如 team-2-laptop。

对话 WebSocket

客户端地址认证

配置 Ollama 和一致的 AGENT_TOKEN 后，使用 node local-agent.js 启动可选本地智能体。模型设置请参阅 backend-v2/CHAT_AGENT_SETUP.md。

04

常见问题

最常见的安装与运行问题。

前端打开后没有实时数据

检查 VITE_API_URL 是否指向正在运行的后端；修改 .env 后重启 Vite；直接打开 /api/data-channels 检查连接；确认 CORS 允许当前前端来源。

发送程序返回 HTTP 404 Route not found

使用准确地址 /api/data-channels/CHANNEL_SLUG。有效 slug 为 csi-wifi、body-vitals、lighting-acoustic、hic 和 gaze-vision。

发送程序返回 HTTP 401 或 403

发送端密钥与后端 DATA_CHANNEL_API_KEY 不一致。通过 X-API-Key 或 Bearer Authorization 发送，不要把密钥写入前端。

CPS 对话显示本地 Ollama 电脑离线

先启动 Ollama，再启动 local-agent.js。检查 WebSocket 地址和本地、云端的 AGENT_TOKEN 是否一致。

云实例超过 512 MB 内存

保持 KOKORO_ENABLED=false，保留每通道字节上限，减少大型 CSI 数据，并缩短通道历史。
